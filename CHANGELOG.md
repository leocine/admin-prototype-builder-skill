# 更新日志

## v2.2.1

- 修复紧凑工具栏中定宽下拉筛选会被搜索框和按钮挤压的问题。
- 下拉筛选现在保持完整宽度，空间不足时由工具栏自然换行，不再显示“全…”等残缺选项。
- 补充设计规范和交付检查项，防止同类布局回归。

## v2.2.0

- 新增紧凑分页标准，统一当前页、总页数、总条数、数字页码、首页尾页和前后翻页的展示。
- 页数较多时支持页码窗口与跳页输入，页数较少时自动精简，窄屏允许分页信息和操作分行。
- 更新页面种子、列表模板、交付检查和自动校验，确保分页信息完整且所有操作可用。

## v2.1.0

- 将运营、客服、审核和订单类列表的默认视觉密度调整为紧凑模式，缩短工具栏、表格行和分页之间的距离。
- 列表筛选改为响应式扁平工具栏：状态 Tabs、搜索框和属性筛选可以同排或换行，不固定具体排版。
- 更新列表页面种子、组件约束、交付检查和自动校验，减少机械 Card 堆叠并统一紧凑表格规格。

## v2.0.10

- 默认业务界面不再显示“原型数据”Badge、状态预览下拉框或其他原型调试控件，避免被误认为正式功能。
- 页面状态覆盖改为通过 `prototypeState` 查询参数等非可见入口检查，默认打开时保持正常业务页面。
- 更新页面种子、交付检查和自动校验，阻止原型辅助控件再次进入页面头部。

## v2.0.9

- 将 README 的安装说明从私有仓库流程调整为公开仓库流程，安装时不再要求 GitHub 登录或仓库授权。
- 更新可直接发给 Codex 的安装提示词，分别说明首次克隆和已有目录同步方式。
- 调整发布验收要求，明确公开 Release 附件应当无需登录即可下载。

## v2.0.8

- 修复 Select 展开层默认挂载到 `body` 后无法命中 `#root` 作用域样式的问题。
- Select 浮层改为挂载到应用根节点，确保背景、边框、圆角、间距、选中标记和主题颜色完整显示。
- 增加 Portal 挂载规则、浮层视觉检查项和发布回归校验，防止同类问题再次出现。

## v2.0.7

- 将 `AGENTS.md` 明确为智能体维护和修改本 Skill 时的操作约束，不再只强调仓库发布。
- 增加开始工作前的阅读顺序、文件职责、规范与实现同步规则以及安全边界。
- 按修改类型细化校验要求，并明确原型执行流程仍由 `SKILL.md` 负责。

## v2.0.6
