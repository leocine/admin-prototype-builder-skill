# 交付检查清单

导出完成后、正式交付前必须执行本清单。

## 业务

- 从页面结构中能够明确看出页面目标和用户角色。
- 筛选项、字段和操作与用户提供的需求一致。
- 已体现启用条件、权限和状态流转规则。
- 危险操作需要确认。
- 演示数据明确为虚构数据，没有捏造生产事实。

## 组件

- 源码从 `@/components/ui/*` 导入真实组件。
- 页面中没有复制 Button、Input、Table、Card 或 Tabs 的实现。
- 每个操作区域只有一个主按钮。
- 每个输入控件都有可见 Label。
- 仅图标控件具有 `title` 和 `aria-label`。
- 状态同时使用文字表达，不只依赖颜色。
- 缺失值显示“—”。
- 逐一展开 Select 等浮层控件，确认背景、边框、圆角、间距、选中标记和层级均完整。
- Portal 浮层位于 `#root` 内，能够继承作用域化样式和主题变量。

## 状态与交互

- 可以检查加载、正常、空数据、失败、禁用和成功状态。
- 默认页面中没有“原型数据”Badge、状态预览下拉框或其他原型调试控件。
- 需要模拟页面状态时，通过 `?prototypeState=loading|empty|error|disabled|success` 等非可见入口检查，不得占用业务页面头部。
- 页面存在相应功能时，查询、重置、排序、分页、Tabs、表单和确认控件均可操作。
- 重置操作不会刷新页面。
- 失败状态提供恢复操作。
- 成功状态说明操作结果和下一状态。

## 布局

- 桌面宽度下能够立即识别信息层级。
- 窄屏下筛选项纵向排列，操作组能够换行。
- Tabs 和 Table 在自身容器中滚动，不引发页面整体横向滚动。
- 不存在文字裁切、控件重叠或无法触达的操作。
- 页面标题没有在 Card 中重复出现。
- Card 形成有意义的边界，没有机械嵌套。
- 需求存在明确统计指标时，统计摘要位于列表标题下、筛选区上方；没有统计需求时不显示占位卡片或虚构指标。
- 统计卡片在桌面端紧凑横排、窄屏自然换列，名称、数值、正负号和单位均完整可读。
- 高频列表使用紧凑密度；工具栏在宽屏合理同排、窄屏自然换行，没有为了同排而压缩可读性。
- 定宽筛选控件不会被同一行的搜索框或按钮挤压，当前选项文字完整可见。
- 表格使用单层边框容器，表头和数据行紧凑清晰，没有“筛选 Card + 表格 Card”的机械堆叠。
- 分页显示当前页、总页数和总条数；数字页码、前后翻页、首页尾页和跳页按数据规模合理出现且可以操作。

## 离线交付物

- 在选定的内置或外部运行时中，`npm run build` 执行成功。
- `export-prototype.mjs` 执行成功。
- `validate-prototype.mjs` 报告零项失败。
- 导出的 HTML 内联包含 CSS 和 JavaScript。
- 不存在外部 CDN、远程字体、远程图片、localhost 或资源文件依赖。
- 文件可以通过 `file://` 直接打开。
- 交付结果同时提供最终 HTML 和 React 源码链接。
